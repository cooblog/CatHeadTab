## 修复 Dock 拖拽排序 bug
- 问题：Dock 中从左往右拖拽时无法正确调整顺序（例如 A→D，A 总是插到 D 前面而非 D 的位置）
- 原因：reorderDesktopItem 先移除 source 再查找 target 索引，当 source 在 target 左侧时，移除 source 导致 target 索引左移一位，但插入时没有补偿这个偏移
- 修复：在移除 source 之前先记录 source 和 target 的原始位置，然后根据同容器内相对位置正确计算插入索引

## 实现背景图云端同步（跨设备）
### 问题
- 用户上传的本地自定义壁纸只保存在 IndexedDB，云端只同步 `idb://` 引用字符串
- 另一台设备拉取云端数据时，引用指向的图片不存在，背景显示为空

### 方案设计
- **存储格式**：WebP（浏览器 Canvas API 压缩），最大 2560×1440，目标 ≤2MB
- **数据库**：新建 `user_backgrounds` 表，使用 BYTEA 存储 WebP 二进制（每用户一张，migration 005）
- **云端标记**：preferences 中 backgroundImage 存为 `cloud://background` 表示图片在云端

### 后端新增
- `backend/migrations/005_create_user_backgrounds.sql` — user_backgrounds 表
- `backend/internal/model/background.go` — UserBackground model
- `backend/internal/repository/background.go` — BackgroundRepository（Upsert/Get/Delete）
- `backend/internal/handler/background.go` — Upload(POST)/Download(GET)/Delete(DELETE) handler
- 路由：`POST/GET/DELETE /api/v1/user/background`
- 上传限制 5MB，支持 multipart/form-data

### 后端 bug 修复
- `repository/user.go` 的 scanRow 缺少 preferences 列的 Scan，导致 GetPreferences 永远返回空
- 所有 SELECT 查询加入 preferences 字段，scanRow 新增 JSON 反序列化

### 前端修改
- `utils/imageStore.ts` — 新增 `compressImageToWebP()`（Canvas 缩放 + 质量递减压缩）、`getRawBlob()`
- `store/layoutStore.ts` — uploadLayoutToCloud 检测 idb:// → 取 Blob → 压缩 WebP → FormData 上传
- `store/layoutStore.ts` — pullLayoutFromCloud 检测 cloud://background → 下载 Blob → 存 IndexedDB
- `components/SettingsModal.tsx` — 上传时先压缩为 WebP，添加 isCompressing 加载动画，限制原始文件 ≤20MB
- i18n 新增 `settings.bgTooLarge`（en/zh）

### 修复：SettingsModal 保存壁纸时未主动上传到云端
- 问题：handleSave 只调用 setBackgroundImage 更新本地状态，没有调用云端上传接口
- 壁纸上传只在 layoutStore.uploadLayoutToCloud（布局变更时的自动同步）中顺带触发
- 修复：handleSave 改为 async，保存后检查 jwtToken，已登录则：
  - idb:// 本地图片 → getRawBlob → compressImageToWebP → POST /api/v1/user/background + PUT preferences 为 cloud://background
  - URL 壁纸 → PUT preferences 同步 URL 字符串
- 引入 client 和 getRawBlob 到 SettingsModal
